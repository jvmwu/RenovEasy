## 贡献指南

当遇到新问题时，请按照以下格式记录：

1. **问题描述** - 详细描述问题现象
2. **错误信息** - 包含完整的错误日志
3. **问题原因** - 分析问题的根本原因
4. **解决方案** - 提供详细的解决步骤
5. **预防措施** - 如何避免类似问题
6. **相关信息** - 版本信息、环境信息等

## 命名规范

文件命名格式：`{问题类型}-{简短描述}.md`

例如：
- `react-native-safe-area-context-crash.md`
- `nativewind-configuration-error.md`
- `ios-build-failure.md`
